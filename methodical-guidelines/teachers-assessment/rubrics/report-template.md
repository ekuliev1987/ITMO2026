# Критерии оценки отчёта по практической работе

## Шаблон рубрикатора

**Примечание:** Этот рубрикатор является шаблоном. Конкретные веса баллов требуют валидации после первого запуска курса.

---

## 1. Цель и постановка задачи (10%)

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| Формулировка цели | Нет цели | Цель общая | Цель конкретная, измеримая |
| Задачи | Нет задач | Перечислены | Декомпозиция на подзадачи |

---

## 2. Описание данных (15%)

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| Источник | Не указан | Указан источник | Источник + ссылка + лицензия |
| Описание | Нет описания | Базовое описание | Структура, размер, статистика |
| Предобработка | Не описана | Перечислены шаги | Обоснование каждого шага |
| Визуализация | Нет | Есть графики | Графики + интерпретация |

---

## 3. Методология (20%)

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| Выбор подхода | Нет обоснования | Обоснование слабое | Обоснование + сравнение альтернатив |
| Архитектура | Не описана | Схема есть | Схема + описание компонентов |
| Гиперпараметры | Не указаны | Указаны значения | Обоснование выбора значений |
| Эксперименты | Один запуск | Несколько запусков | Статистика по нескольким запускам |

---

## 4. Результаты (25%)

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| Метрики | Нет метрик | Метрики есть | Метрики + сравнение с baseline |
| Таблицы/графики | Нет | Есть | Наглядные + подписанные |
| Воспроизводимость | Нет деталей | Есть детали | Полные детали для повтора |
| Сопоставление с целью | Нет | Частичное | Полное соответствие цели |

---

## 5. Анализ ошибок (20%) **[КРИТИЧНО ДЛЯ ML]**

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| Примеры ошибок | Нет | Есть примеры | Примеры + классификация ошибок |
| Гипотезы | Нет | Есть | Гипотезы + проверка |
| Ограничения | Не указаны | Указаны | Ограничения + влияние на применение |

---

## 6. Выводы (10%)

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| Итоги | Нет выводов | Выводы формальные | Конкретные выводы по задаче |
| Дальнейшие шаги | Нет | Есть | Конкретные предложения улучшений |

## Рекомендуемая структура отчёта

```
1. Титульный лист
   - Название модуля и задания
   - ФИО студента, группа, дата

2. Цель и задачи
   - Что должно было быть сделано
   - Критерии успеха

3. Данные
   - Источник и описание
   - Предобработка (с кодом/скриншотами)
   - Визуализация распределений

4. Метод
   - Выбор архитектуры/алгоритма (с обоснованием)
   - Схема/псевдокод
   - Гиперпараметры и их настройка

5. Эксперименты
   - Описание экспериментальной установки
   - Результаты (таблицы, графики)
   - Сравнение с baseline/другими подходами

6. Анализ ошибок
   - Типичные ошибки модели
   - Примеры (успешные и неудачные)
   - Гипотезы о причинах

7. Выводы
   - Достигнута ли цель
   - Что сработало, что нет
   - Предложения по улучшению

8. Приложения
   - Ссылка на код (GitHub/GitLab)
   - Дополнительные графики/таблицы
```

---

## Чек-лист для быстрой проверки

**Обязательные элементы:**
- [ ] Цель сформулирована конкретно
- [ ] Источник данных указан
- [ ] Метрики качества приведены
- [ ] Есть анализ ошибок
- [ ] Ссылка на код работает

**Качество:**
- [ ] Графики подписаны (оси, легенда)
- [ ] Таблицы имеют заголовки
- [ ] Обоснование выбора подхода
- [ ] Выводы соответствуют результатам

**Воспроизводимость:**
- [ ] Можно повторить эксперимент по отчёту
- [ ] Указаны версии библиотек
- [ ] Гиперпараметры зафиксированы

## Специфика модулей

### M1 (EDA, визуализация)
- Акцент на качество визуализаций
- Проверка: осмысленность выводов из EDA

### M2-M3 (VAE, GAN)
- Акцент на анализ качества генерации
- Проверка: метрики (FID, MSE), примеры сгенерированных данных

### M4 (LLM, Prompt Engineering)
- Акцент на сравнение промптов
- Проверка: таблицы результатов для разных промптов

### M5 (RAG, мультимодальность)
- Акцент на метрики RAG (faithfulness, precision)
- Проверка: примеры вопросов и ответов

### M6 (Deployment)
- Акцент на документацию API
- Проверка: Swagger/OpenAPI спецификация
