# Критерии оценки качества кода

## Шаблон рубрикатора

**Примечание:** Этот рубрикатор является шаблоном. Конкретные веса баллов требуют валидации после первого запуска курса.

---

## 1. Работоспособность (40%)

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| Запуск | Код не запускается | Запускается с ошибками | Запускается без ошибок |
| Результат | Не достигает цели | Частичный результат | Полный результат |
| Обработка ошибок | Нет обработки | Базовая обработка | Полная обработка (edge cases) |

---

## 2. Воспроизводимость (25%) **[КРИТИЧНО ДЛЯ ML]**

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| Случайные числа | Нет фиксации seed | Seed только в одном месте | Seed везде + логирование |
| Зависимости | Нет списка | requirements.txt есть | requirements.txt + версии |
| Конфигурация | Хардкод параметров | Конфиг в отдельном файле | Конфиг + валидация параметров |
| Данные | Путь зашит в коде | Путь в конфиге | Путь в конфиге + проверка наличия |

---

## 3. Структура и модульность (15%)

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| Разделение ответственности | Всё в одном файле | Частичное разделение | Чёткое разделение (data/model/train/eval) |
| Переиспользование | Дублирование кода | Минимальное дублирование | DRY — нет повторений |
| Импорты | Хаотичные импорты | Группировка по типу | Чёткая структура импортов |

---

## 4. Документация и читаемость (15%)

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| Именование | Непонятные имена (a, b, x1) | Смешанный стиль | Семантически понятные имена |
| Комментарии | Нет или избыток | Минимальные | Поясняют «почему», а не «что» |
| docstring | Нет описания функций | Есть описания | Полные docstring с примерами |
| README | Нет README | Есть README | README с инструкцией запуска |

---

## 5. Лучшие практики (5%)

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| PEP8 | Нарушения | Минимальные нарушения | Соответствует PEP8 |
| Типизация | Нет типов | Частичная типизация | Полная типизация (type hints) |
| Логирование | print() везде | Частичное логирование | logging + уровни логов |

## Чек-лист для быстрой проверки

**Воспроизводимость (обязательно):**
- [ ] `random.seed()` / `torch.manual_seed()` / `np.random.seed()` установлены
- [ ] `requirements.txt` или `environment.yml` существует
- [ ] Можно запустить код с чистого листа по README

**Структура:**
- [ ] Нет файлов > 300 строк без разделения
- [ ] Конфигурация вынесена из кода
- [ ] Данные и код разделены

**Документация:**
- [ ] README содержит инструкцию «как запустить»
- [ ] Критические функции имеют docstring
- [ ] Имена переменных понятны без комментариев

## Специфика модулей

### M1 (Python, NumPy, Pandas)
- Акцент на структуру данных и векторизацию
- Проверка: нет ли циклов там, где можно использовать numpy

### M2-M3 (VAE, GAN)
- Акцент на воспроизводимость экспериментов
- Проверка: сохранены ли веса модели, лог метрик

### M4-M5 (LLM, RAG)
- Акцент на конфигурацию моделей и токенизаторов
- Проверка: зафиксированы ли версии моделей с Hugging Face

### M6 (Deployment)
- Акцент на Dockerfile, FastAPI структуру
- Проверка: можно ли запустить через `docker-compose up`
