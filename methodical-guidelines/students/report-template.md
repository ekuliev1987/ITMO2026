# Шаблон отчёта по практической работе

> Используйте этот шаблон для оформления отчётов по практическим работам.

---

## Общие требования

- Формат: Markdown (`.md`) или PDF
- Объём: 2-5 страниц (в зависимости от модуля)
- Язык: русский или английский
- Код: выносится в отдельный файл/репозиторий, в отчёте — только ключевые фрагменты

---

## Структура отчёта

### 1. Титульная информация

```markdown
# Модуль X: Название модуля

**Студент:** ФИО, группа  
**Дата:** ДД.ММ.ГГГГ  
**Модуль:** M1/M2/M3/M4/M5/M6
```

### 2. Цель и задачи

```markdown
## Цель работы

[Кратко опишите, что должно было быть сделано]

## Задачи

1. [Задача 1]
2. [Задача 2]
3. [Задача 3]
```

**Что писать:**
- Чёткая формулировка цели
- Конкретные измеримые задачи
- Критерии успеха

---

### 3. Данные

```markdown
## Источник данных

- Название датасета:
- Ссылка:
- Лицензия:
- Размер (количество записей, размер файла):

## Предобработка

- Пропущенные значения: [как обработали]
- Дубликаты: [удалены/оставлены]
- Нормализация/стандартизация: [да/нет, как]
- Разделение на train/test: [соотношение]

## Визуализация данных

![Описание графика](путь_к_графику.png)

*Краткое описание, что показывает график*
```

**Что писать:**
- Полный источник данных (ссылка, лицензия)
- Все шаги предобработки с обоснованием
- Визуализация распределений/статистики

---

### 4. Метод

```markdown
## Выбор подхода

[Почему выбрали именно этот метод/архитектуру?]

## Архитектура модели

[Описание архитектуры, схема при необходимости]

```python
# Ключевой фрагмент кода
model = ...
```

## Гиперпараметры

| Параметр | Значение | Обоснование |
|----------|----------|-------------|
| learning_rate | 0.001 | Стандартное значение |

- Оборудование: [GPU/CPU, модель]
- Время обучения: [часы/минуты]
```

**Что писать:**
- Обоснование выбора метода (сравнение альтернатив)
- Схема/описание архитектуры
- Все гиперпараметры с обоснованием
- Условия эксперимента

```markdown

| Метрика | Значение | Baseline |
|---------|----------|----------|

| F1-score | 0.84 | 0.72 |
| ... | ... | ... |

## Визуализация результатов

![График результатов](путь_к_графику.png)

*Интерпретация результатов*

## Сравнение с baseline

[Как ваши результаты сравниваются с базовым подходом?]
```

**Что писать:**
- Таблицы с метриками
- Графики с подписанными осями
- Сравнение с baseline/другими подходами
- Интерпретация результатов

**Что писать:**
- Конкретные примеры ошибок

- Гипотезы о причинах

## Чек-лист перед отправкой

**Обязательные элементы:**
- [ ] Цель сформулирована конкретно
- [ ] Источник данных указан с лицензией
- [ ] Метрики качества приведены
- [ ] Есть анализ ошибок с примерами
- [ ] Ссылка на код работает

**Качество:**
- [ ] Графики подписаны (оси, легенда)
- [ ] Таблицы имеют заголовки
- [ ] Обоснование выбора подхода
- [ ] Выводы соответствуют результатам

**Воспроизводимость:**
- [ ] Можно повторить эксперимент по отчёту
- [ ] Указаны версии библиотек
- [ ] Гиперпараметры зафиксированы

## Примеры заполнения

**Хороший пример:**
- Цель: "Увеличить F1-score на 5% по сравнению с baseline"
- Данные: "Hugging Face Datasets, license: MIT, 10K записей"
- Метод: "Выбран BERT-base, так как показывает SOTA на подобных задачах"
- Результаты: "F1=0.87 vs baseline F1=0.82 (+6%)"
- Анализ: "Модель ошибается на коротких текстах (<10 слов), гипотеза: недостаточный контекст"

**Плохой пример:**
- Цель: "Сделать модель"
- Данные: "Взял откуда-то"
- Метод: "Использовал нейросеть"
- Результаты: "Получил хорошие результаты"
- Анализ: "Ошибок нет"

## Специфика модулей

### M1 (EDA, визуализация)
- Акцент на качество визуализаций
- Выводы из EDA (что обнаружили в данных)

### M2-M3 (VAE, GAN)
- Метрики генерации (FID, MSE)
- Примеры сгенерированных данных
- Интерполяция в латентном пространстве

### M4 (LLM, Prompt Engineering)
- Сравнение разных промптов
- Таблицы результатов

### M5 (RAG, мультимодальность)
- Метрики RAG (faithfulness, relevancy)
- Примеры вопросов и ответов

### M6 (Deployment)
- API спецификация (Swagger/OpenAPI)
- Тесты на производительность
