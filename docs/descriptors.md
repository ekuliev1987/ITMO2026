# Дескрипторы уровней освоения компетенций

## Определение уровней

| Уровень | Код | Описание | Что студент может |
|---------|-----|----------|-------------------|
| **Базовый** | Б | Знание терминов, понимание концепций, воспроизведение по шаблону | • Дать определения ключевым терминам<br>• Описать принцип работы метода<br>• Выполнить типовое задание по образцу<br>• Использовать готовые библиотеки с минимальной модификацией |
| **Средний** | С | Понимание принципов, адаптация к новым задачам | • Объяснить выбор подхода для конкретной задачи<br>• Адаптировать решение под новые условия<br>• Настроить гиперпараметры модели<br>• Оценить качество решения и выявить проблемы<br>• Интегрировать несколько компонентов в пайплайн |
| **Продвинутый** | П | Создание новых решений, оптимизация, обоснование выбора | • Разработать новое решение или модификацию<br>• Оптимизировать производительность системы<br>• Обосновать выбор архитектуры/алгоритма<br>• Провести сравнительный анализ подходов<br>• Решать нестандартные задачи |

---

## Примеры дескрипторов для дисциплины

### ML-2: Машинное обучение

| Уровень | Дескриптор |
|---------|------------|
| Б | Различает типы задач ML (классификация, регрессия, кластеризация), знает основные алгоритмы |
| С | Может выбрать подходящий алгоритм для задачи, подготовить данные, оценить качество модели |
| П | Может улучшить существующие алгоритмы, создать ансамбли, провести глубокий анализ ошибок |

### DL-2: Глубокое обучение

| Уровень | Дескриптор |
|---------|------------|
| Б | Понимает архитектуру нейронных сетей, принцип обратного распространения ошибки |
| С | Может реализовать архитектуру (CNN, RNN, GAN, VAE), настроить обучение, стабилизировать процесс |
| П | Может разработать новую архитектуру, провести нейроархитектурный поиск, оптимизировать для production |

### LLM-5: Большие языковые модели

| Уровень | Дескриптор |
|---------|------------|
| Б | Знает принцип работы трансформеров, может использовать готовые модели через API |
| С | Может построить RAG-систему, дообучить модель (LoRA), создать AI-агента |
| П | Может разработать новую архитектуру внимания, оптимизировать инференс, создать фреймворк для агентов |

### PL-1: Программирование для ML

| Уровень | Дескриптор |
|---------|------------|
| Б | Владеет базовыми инструментами Python (NumPy, Pandas), может загрузить и визуализировать данные |
| С | Может реализовать пайплайн обработки данных, написать модульный код, использовать Git |
| П | Может разработать библиотеку/фреймворк, оптимизировать производительность, развернуть сервис |

### FC-2: Фундаментальные компетенции

| Уровень | Дескриптор |
|---------|------------|
| Б | Понимает математическую основу методов (линейная алгебра, теория вероятностей, оптимизация) |
| С | Может вывести формулы, обосновать выбор функции потерь, проанализировать сходимость |
| П | Может разработать новый метод оптимизации, доказать свойства алгоритма |

### SS-1: Социальные и этические аспекты

| Уровень | Дескриптор |
|---------|------------|
| Б | Знает основные этические риски ИИ (bias, deepfakes, авторское право) |
| С | Может провести этический аудит системы, предложить меры по снижению рисков |
| П | Может разработать методологию этической оценки, создать фреймворк для ответственного ИИ |
