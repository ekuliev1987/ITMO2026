# Вклад в проект

## Как внести вклад

1. Создайте issue для обсуждения изменения
2. Fork репозитория
3. Создайте ветку: `feature/ваше-изменение`
4. Внесите изменения
5. Протестируйте изменения
6. Создайте Pull Request

## Стиль кода

- Используйте Python 3.10+
- Следуйте PEP 8
- Добавляйте docstrings к функциям и классам
- Пишите тесты для нового кода

## Формат коммитов

```
type(scope): description

[optional body]
```

Типы коммитов:
- `feat`: новая функциональность
- `fix`: исправление ошибки
- `docs`: документация
- `style`: форматирование
- `refactor`: рефакторинг
- `test`: тесты
- `chore`: обслуживание

## Проверка перед отправкой

- [ ] Код проходит linting
- [ ] Добавлены тесты (если применимо)
- [ ] Обновлена документация
- [ ] Нет лишних файлов
