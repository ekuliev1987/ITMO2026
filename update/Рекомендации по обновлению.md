# Рекомендации по обновлению рабочей программы дисциплины
## «Генеративный искусственный интеллект и машинное обучение»

---

## 1. Анализ текущей программы

### Сильные стороны:
- ✅ Comprehensive coverage of foundational topics (VAE, GAN, Transformers, Diffusion)
- ✅ Well-structured practical assignments with basic/advanced levels
- ✅ Strong theoretical foundation with exam questions
- ✅ Good emphasis on ethics and responsible AI
- ✅ Progressive difficulty from basic ML to advanced generative models

### Выявленные проблемы:

#### 1.1. **Устаревшие темы и отсутствие актуальных направлений**

**Проблема:** Программа не включает ключевые современные технологии 2024-2026 гг.:

| Отсутствующая тема | Важность | Рекомендация |
|---|---|---|
| **Large Language Models (LLM) архитектуры** | Высокая | Добавить Llama, Mistral, Qwen архитектуры |
| **Parameter-Efficient Fine-Tuning (PEFT)** | Высокая | LoRA, QLoRA, adapters — критически важно для практики |
| **RAG (Retrieval-Augmented Generation)** | Высокая | Векторные базы данных, LangChain, LlamaIndex |
| **Multimodal Models нового поколения** | Высокая | CLIP, BLIP, Flamingo, GPT-4V, Gemini |
| **Diffusion Models продвинутые** | Средняя | ControlNet, DreamBooth, LoRA для диффузии |
| **AI Agents и автономные системы** | Высокая | ReAct, Tool Use, AutoGen, LangGraph |
| **Model Quantization и оптимизация** | Средняя | INT8/INT4, GGUF, vLLM, TGI |
| **MLOps для генеративных моделей** | Средняя | MLflow, Weights & Biases, Hugging Face Hub |

#### 1.2. **Компетентностно-ролевая модель — несоответствие**

Согласно современной компетентностно-ролевой модели в области ИИ, выпускник должен быть способен занимать следующие роли:

| Роль | Необходимые компетенции | Текущее покрытие |
|---|---|---|
| **AI Engineer** | Deployment, API, Production | ⚠️ Частично |
| **ML Engineer** | Training, Fine-tuning, Evaluation | ✅ Хорошо |
| **Prompt Engineer** | Prompt Design, Optimization | ⚠️ Базово |
| **AI Researcher** | Novel architectures, Papers | ⚠️ Теоретически |
| **AI Ethicist** | Bias detection, Fairness | ✅ Хорошо |
| **AI Product Manager** | Use cases, ROI, Strategy | ❌ Отсутствует |

#### 1.3. **Практическая часть — недостатки**

- Нет работы с **векторными базами данных** (Chroma, Pinecone, Weaviate)
- Отсутствует **развёртывание моделей** (Docker, FastAPI, Gradio)
- Нет **оценки качества генеративных моделей** (BLEU, ROUGE, BERTScore, CLIP Score, FID)
- Отсутствует **оптимизация инференса** для production
- Нет **сравнительного анализа моделей** на реальных датасетах

#### 1.4. **Литература и ресурсы**

**Проблемы:**
- Часть источников устарела (2020-2022 гг.)
- Отсутствуют ссылки на ключевые papers 2023-2025 гг.
- Нет ссылок на интерактивные платформы (Hugging Face Courses, DeepLearning.AI)

### 2.2. **Обновить практические задания**

#### Добавить новые практические работы:

**Практическое занятие 10: RAG-системы и векторные базы данных**
- Цель: Построить систему вопрос-ответ на основе собственных документов
- Задачи:
  - Работа с LangChain / LlamaIndex
  - Использование векторных БД (Chroma, FAISS)
  - Оценка качества ответов (RAGAS)

**Практическое занятие 11: Parameter-Efficient Fine-Tuning**
- Цель: Дообучить LLM с помощью LoRA/QLoRA
- Задачи:
  - Использование библиотеки peft
  - Обучение на кастомном датасете
  - Сравнение с полным fine-tuning

**Практическое занятие 12: Развёртывание генеративных моделей**
- Цель: Создать production-ready API для генеративной модели
- Задачи:
  - Обёртывание модели в FastAPI
  - Контейнеризация с Docker
  - Интеграция с Gradio/Streamlit UI

**Практическое занятие 13: AI Agents и автономные системы**
- Цель: Создать агента с возможностью использования инструментов
- Задачи:
  - Реализация ReAct-архитектуры
  - Интеграция с внешними API
  - Использование LangGraph/AutoGen

## 5. Заключение

Предложенные обновления обеспечат:
- **Актуальность** программы с учётом технологий 2024-2026 гг.
- **Практическую применимость** для трудоустройства в индустрию

Рекомендуется внедрить обновления в два этапа:
1. **Этап 1 (критические изменения):** PEFT, RAG, современные LLM — 1-2 месяца
2. **Этап 2 (дополнительные улучшения):** Agents, MLOps, Multimodal — 3-6 месяцев
